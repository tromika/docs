# üïµÔ∏è Cantina Code for Security Researchers

The Cantina Code interface allows Security Researchers to submit [findings](findings/ "mention") and communicate with clients or any other members of your [teams.md](../../cantina-competitions/for-security-researchers/teams.md "mention") inside a Cantina Repository.

In this section we will outline some of the most important features and _"must knows" of_ the interface.

<figure><img src="../../.gitbook/assets/spaces_CQ6xUmZJe2xLEB2WIGr2_uploads_YR4APzfFmUBNCHgu5MsW_image.png" alt="Smart contract audit findings interface."><figcaption><p>Cantina Code Repository Interface</p></figcaption></figure>

On the **top right side** of the interface you have access to the [cantina-account](../../cantina-account/ "mention"), [Broken link](broken-reference "mention") inbox and **cantina** **repository information,** which contain information such as:

* **Type**: The type of the security engagement. e.g., Public Competition, Collaborative review, etc..
* **Status**: Different statuses that affect repository settings. e.g., Live, Judging, Escalations, Completed, etc..
* **Time-frame**: Duration of the security engagement.
* **Role**: User role in the repository.

<figure><img src="../../.gitbook/assets/image (1).png" alt="Security audit status header."><figcaption><p>Repository information</p></figcaption></figure>

and on the **top left side** of the interface is the **name of the repository**, e.g., `Morpho/moprho-blue`.

<figure><img src="../../.gitbook/assets/image.png" alt="Protocol review identifier."><figcaption></figcaption></figure>
